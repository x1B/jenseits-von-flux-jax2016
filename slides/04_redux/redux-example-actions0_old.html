Redux - Beispiel - Actions (1/3)


import shop from '../../../common/api/shop';

// product actions
export const RECEIVE_PRODUCTS = 'RECEIVE_PRODUCTS';
function receiveProducts(products) {
    return {
        type: RECEIVE_PRODUCTS,
        products: products
    };
}
export function getAllProducts() {
    return dispatch => {
        shop.getProducts(products => {
            dispatch(receiveProducts(products));
        });
    };
}

// ...
- Oben sehen wir einen gewöhnlichen Action-Creator - Dieser liefert nur ein einfaches Objekt zurück - Durch `thunk` ist es möglich, dass Action Creators nicht nur auf Objekte beschränkt sind - Stattdessen können auch Funktionen an `dispatch` übergeben werden, die selber wieder `dispatch` und eine `getState` Funktion erhalten - Diese Action-Creator sind selbst aber erstmal auch reine Funktionen ohne Seiteneffekte - Hier wird `dispatch` verwendet um auf Basis einer asynchronen API eine Action zu dispatchen - Die Produktliste wird asynchron abgerufen und anschließend die RECEIVE_PRODUCTS-Action dispatched - da der receiveProducts Action-Creator ausserhalb nicht sinnvoll eingesetzt werden kann, wird dieser hier auch gar nicht exportiert - Nur der Action-Typ wird exportiert, da der bei den Reducern interessant wird -> sehen wir aber später
  • Action Creators erzeugen reine Action-Objekte
  • Dispatch durch View oder Middleware (hier thunk)
  • thunk: Funktion mit dispatch, state als Parameter
  • für asynchrone Operationen
